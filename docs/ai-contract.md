# AI Contract

This document records the Spring server boundary with external AI services. Values marked TODO still require AI team confirmation.

## Common Headers

- `X-Correlation-Id`: generated by the Spring server when absent and forwarded to AI.
- `X-API-Key`: sent from Spring to AI only when `AI_API_KEY` is configured.

The RN app calls Spring with `X-Installation-Id`. This is an installation UUID, not authentication. Spring stores only a SHA-256 hash of this value in `AiAnalysisJob`.

## Runtime Modes

- `AI_MODE=mock`: use mock clients.
- `AI_MODE=real`: use HTTP clients. `AI_BASE_URL` is required.
- `AI_FITNESS_MODE`: optional override for fitness analysis.
- `AI_POSTURE_MODE`: optional override for posture analysis.

Timeouts:

- `AI_CONNECT_TIMEOUT`
- `AI_READ_TIMEOUT`

Retry:

- `AI_RETRY_COUNT` applies only to retryable fitness JSON requests and transient server/network failures.
- Posture video uploads are not blindly retried.

## Fitness Analysis

Spring endpoint:

- `POST /api/fitness-analyses`

AI HTTP endpoint:

- `${AI_BASE_URL}${AI_FITNESS_PATH}`

Transport:

- `application/json`

Request:

```json
{
  "correlationId": "uuid",
  "modelVersion": null,
  "profile": {
    "birthDate": "2010-05-01",
    "gender": "MALE",
    "schoolLevel": "HIGH",
    "schoolGrade": 1,
    "heightCm": 170.5,
    "weightKg": 60.2
  },
  "records": [
    {
      "itemCode": "PUSH_UP",
      "value": 20,
      "unit": "COUNT",
      "measuredAt": "2026-07-01T00:00:00Z"
    }
  ]
}
```

Response:

```json
{
  "correlationId": "uuid",
  "modelVersion": "fitness-model-v1",
  "summary": "text",
  "recommendations": [
    {
      "title": "text",
      "description": "text"
    }
  ]
}
```

## Posture Analysis

Spring endpoint:

- `POST /api/posture-analyses`

AI HTTP endpoint:

- `${AI_BASE_URL}${AI_POSTURE_PATH}`

Transport:

- `multipart/form-data`

Parts:

- `exerciseType`: text, one of `PUSH_UP`, `SQUAT`, `LUNGE`, `PLANK`
- `video`: binary video file

Spring-generated multipart filenames are safe placeholders. Original client filenames are not forwarded.

The included `ai-python` FastAPI service endpoint is `/pose/extract`. It extracts pose metrics and returns metric JSON. Spring adapts that metric response to the existing posture analysis response DTO.

```json
{
  "exerciseType": "SQUAT",
  "durationSec": 1.2,
  "sampledFps": 10,
  "personDetected": true,
  "metrics": {}
}
```

Per-exercise `metrics` shape (all computed by `ai-python`, pure computer vision, no LLM involved):

- `PUSH_UP`: `{repCount, reps: [{startSec, endSec, minElbowAngleDeg, maxHipSagDeg, neckAlignedRatio}], bodyLineDeviationSeries: [{sec, deg}]}`
- `SQUAT`: `{repCount, reps: [{startSec, endSec, minKneeAngleDeg, maxTrunkLeanDeg}]}`
- `LUNGE`: `{repCount, reps: [{startSec, endSec, minFrontKneeAngleDeg, maxTrunkLeanDeg}]}`
- `PLANK` (a held posture, not reps): `{holdSec, maxHipSagDeg, maxHipPikeDeg, bodyLineDeviationSeries: [{sec, deg}]}`

Response:

```json
{
  "correlationId": "uuid",
  "modelVersion": "posture-model-v1",
  "status": "COMPLETED",
  "feedback": [
    {
      "code": "KNEE_ALIGNMENT",
      "message": "text",
      "severity": "MEDIUM"
    }
  ]
}
```

## Job Status

Spring stores temporary `AiAnalysisJob` state:

- `PENDING`
- `PROCESSING`
- `COMPLETED`
- `FAILED`
- `EXPIRED`

Request and result payloads are temporary lookup data and are cleared after expiration. They are not long-term user history.

## Video Storage

Spring stores posture videos only while forwarding analysis requests. Temporary files are deleted after AI success, AI 4xx/5xx, timeout, invalid response, or unexpected runtime failure. Logs must not contain original filenames, local temp paths, full AI payloads, or installation IDs.
